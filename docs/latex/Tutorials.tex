E\+C\+I\+QC is a flexible framework for medical imaging quality control. Users can teach it what to do by writing configuration files. In this tutorial, we are going to introduce how to specify proper configuration files at successively increasing levels of details. This tutorial is suitable for both non-\/developers (end-\/users) and developers new to this software. At the end of this tutorial, you will learn
\begin{DoxyItemize}
\item General syntaxes of configuration files;
\item How to modify the example configuration files to cater to users\textquotesingle{} needs;
\item How to start from strach to write proper configuration files.
\end{DoxyItemize}

Here are some common scenarios proposed by Chris from GE Healthcare. You can find the corresponding configuration codes in configuration gallery page.
\begin{DoxyItemize}
\item Case1\+: check whether the patient name is a 6-\/digit number. If so, insert the patient name to clinical trial subject id; reject otherwise.
\item Case2\+: check whether the clinical trial protocol id is 3 letters followed by 6 digits. If so, prepend the patient name with \char`\"{}\+\_\+\char`\"{} and clinical trial protocol id, reject otherwise.
\item Case3\+: check whether the date of birth is a 8-\/digit number. Replace the day with 30 and month with 06 if so.
\item Case4\+: check whether the modality is \textquotesingle{}MR\textquotesingle{}. If so, check whether the manufacturer is \textquotesingle{}S\+I\+E\+M\+E\+NS\textquotesingle{} or \textquotesingle{}H\+I\+T\+A\+C\+HI\textquotesingle{}. If that\textquotesingle{}s true, check whether the magnetic field strenght equals 1.\+5 or 3.\+0. Set the clinical trial series description to T1 if all these conditions are satisfied; reject otherwise.

\hypertarget{Tutorials_autotoc_md13}{}\doxysubsubsection{Explanation}\label{Tutorials_autotoc_md13}
The E\+C\+I\+QC receives medical images via {\bfseries{receiver}} and sends post-\/processed images via {\bfseries{sender}}.The key {\ttfamily receiver} specifies the port number so that another remote sender can identify it and send proper images. The E\+C\+I\+QC pipeline checks the image and send normal images to the {\bfseries{destination}} and suspicious ones to the {\bfseries{quarantine}}. {\ttfamily Destination} and {\ttfamily quarantine} tell the E\+C\+I\+QC their addresses.

The key {\ttfamily metadata} is the main component of the configuration file. You will see long lines of code within this part. Typically, you will see one or more D\+I\+C\+OM tags to work on(0010,0010), accompanied by the tag name (Patient Name),vr (data type and format) and a short line of description. The key {\ttfamily operations} inform the E\+C\+I\+QC what to do with this tag. You can guess with your gut instinct that this configuration file requires the pipeline to replace the name with Robert if the patient name is John Doe.\hypertarget{Tutorials_autotoc_md14}{}\doxysubsection{Receiver, Destination and Quarantine}\label{Tutorials_autotoc_md14}
As said before, {\ttfamily receiver} specifies the proper of the E\+C\+I\+QC receiver; {\ttfamily destination} tells the E\+C\+I\+QC where to send \char`\"{}good\char`\"{} images; {\ttfamily quarantine} provides the place to send \char`\"{}bad\char`\"{} images. You can use the local port for most test cases. However,if you are interested in communicating between different network nodes in real-\/world application (e.\+g.
