E\+C\+I\+QC is a flexible framework for medical imaging quality control. Users can teach it what to do by writing configuration files. In this tutorial, we are going to introduce how to specify proper configuration files at successively increasing levels of details. This tutorial is suitable for both non-\/developers (end-\/users) and developers new to this software. At the end of this tutorial, you will learn
\begin{DoxyItemize}
\item General syntaxes of configuration files;
\item How to modify the example configuration files to cater to users\textquotesingle{} needs;
\item How to start from strach to write proper configuration files.
\end{DoxyItemize}

Here are some common scenarios proposed by Chris from GE Healthcare. You can find the corresponding configuration codes in configuration gallery page.
\begin{DoxyItemize}
\item Case1\+: check whether the patient name is a 6-\/digit number. If so, insert the patient name to clinical trial subject id; reject otherwise.
\item Case2\+: check whether the clinical trial protocol id is 3 letters followed by 6 digits. If so, prepend the patient name with \char`\"{}\+\_\+\char`\"{} and clinical trial protocol id, reject otherwise.
\item Case3\+: check whether the date of birth is a 8-\/digit number. Replace the day with 30 and month with 06 if so.
\item Case4\+: check whether the modality is \textquotesingle{}MR\textquotesingle{}. If so, check whether the manufacturer is \textquotesingle{}S\+I\+E\+M\+E\+NS\textquotesingle{} or \textquotesingle{}H\+I\+T\+A\+C\+HI\textquotesingle{}. If that\textquotesingle{}s true, check whether the magnetic field strenght equals 1.\+5 or 3.\+0. Set the clinical trial series description to T1 if all these conditions are satisfied; reject otherwise.
